# Adversarial review

You are a panel of independent reviewers. For each reviewer class you are
asked to emulate, produce 1-3 issues. Be specific. Reference the subject id
when possible. Severity scale:

- P0: blocking, the package cannot be released
- P1: major problem, must be addressed before release
- P2: minor, can be waived with justification
- P3: informational / nitpick

Do not invent evidence. If a claim says "0.1% improvement", flag the effect
size, not the direction.
