# Experiment specification

Turn an idea into a concrete, minimal, falsifiable experiment. The spec must
be reproducible by a careful engineer without follow-up questions.

Required:
- hypothesis: one sentence, testable
- problem_framing: 2-4 sentences, what is the question and why it is answerable
- target_metrics: at least one, with names operational engineers recognise
- dataset_assumptions: which dataset, which splits, how it is obtained
- baseline: exact baseline we will compare against
- experiment_plan: step-by-step; someone else should be able to follow it
- constraints: compute, time, dependencies
- success_criteria / stop_criteria: measurable
- budget_estimate_usd: optional, conservative
